# Implementation Plan: HermesMacOS Test Target

**Branch**: `main` | **Date**: 2026-06-27 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/013-hermesmacos-test-target/spec.md` plus user constraints: latest Swift toolchain, Xcode 26.6, macOS 26.0 and above.

## Summary

Add a native Xcode/XcodeGen test target named exactly `HermesMacOSTest` and use it to establish complete-scope Swift test coverage for HermesMacOS. The implementation updates the generated project source of truth, moves project language settings to Swift 6 language mode under Xcode 26.6 / Apple Swift 6.3.3, adds deterministic mock-backed functional and technical tests, and documents opt-in live smoke checks for Hermes services and macOS permissions.

## Technical Context

**Language/Version**: Swift 6 language mode using Xcode 26.6 with Apple Swift 6.3.3 (`xcrun swift --version` reported `swiftlang-6.3.3.1.1`; target `arm64-apple-macosx26.0`). Current `project.yml` still sets `SWIFT_VERSION: 5.0`, so implementation must update project and test settings deliberately.  
**Primary Dependencies**: Apple platform frameworks already used by HermesMacOS (SwiftUI, Foundation, Observation, Security, LocalAuthentication, CryptoKit, WebKit, AVFoundation, Speech, AppKit, UniformTypeIdentifiers), XcodeGen 2.45.4, Xcode build/test runner, Swift Testing or XCTest from the Apple toolchain. No third-party runtime dependency is planned.  
**Storage**: Test fixtures under `HermesMacOSTest/Fixtures/`; temporary directories for local Hermes home/repo/config tests; fake Keychain labels or dependency seams for secret tests; no writes to Laurent's real Hermes home, repositories, certificate pins, or production Keychain items during default tests.  
**Testing**: New `HermesMacOSTest` target with default mock-backed functional and technical Swift tests; `xcodegen generate`; `xcodebuild -project HermesMacOS.xcodeproj -scheme HermesMacOS -destination 'generic/platform=macOS' build`; `xcodebuild -project HermesMacOS.xcodeproj -scheme HermesMacOSTest -destination 'platform=macOS' test`; optional live-smoke checklist for configured Hermes services only.  
**Target Platform**: Native macOS 26.0 and above.  
**Project Type**: Desktop SwiftUI app / native Hermes Agent control surface generated by XcodeGen.  
**Performance Goals**: Default test suite remains deterministic and reasonably fast; mock streams and fixtures stay bounded; long-running loops are driven by injectable clocks/tasks or explicit stop conditions; no live network wait is part of default tests.  
**Constraints**: Preserve unsandboxed-app guardrails; block remote plaintext secret traffic; keep credentials redacted; do not mutate real local runtime files; preserve Hermes API/Dashboard/TUI Gateway contracts; avoid substantial growth in high-churn feature files except for small testability seams.  
**Scale/Scope**: One application target plus one new test target; complete coverage map across all documented tabs, Settings, Hermes API, Dashboard, TUI Gateway, local runtime, utilities, speech, retention, TLS, filesystem/process, Git/SSH, and security helper categories.

## Constitution Check

*GATE: Passed before Phase 0 research and re-checked after Phase 1 design.*

- **Native control surface**: Pass. The feature adds project/test infrastructure for the existing native SwiftUI control surface. No new web wrapper or alternate runtime is introduced.
- **Integration contracts**: Pass. Tests explicitly preserve Hermes API (`/v1/responses`, `/v1/chat/completions`, profiles, approvals, cancellation), Dashboard session-token/API routes, TUI Gateway JSON-RPC/WebSocket events, local Hermes files, Git/SSH, speech-to-text, and macOS service contracts through mocks, fixtures, and opt-in live smoke checks.
- **Security guardrails**: Pass. The plan requires fake credentials, redaction assertions, temporary runtime fixtures, endpoint validation tests, TLS fixture fingerprints, local filesystem approval checks, bounded process execution tests, and no default mutation of real Hermes/user state.
- **Verification**: Pass. Implementation must regenerate the project, build `HermesMacOS`, run `HermesMacOSTest`, run focused fixture/coverage verifiers, and document optional live-service smoke checks.
- **Maintainability**: Pass. New tests live under `HermesMacOSTest/`; production changes are limited to `project.yml`, regenerated `.xcodeproj`, focused testability seams, and docs. Large files (`HermesModelsAPI.swift`, `HermesKanbanView.swift`, `HermesViews.swift`, `HermesHistoryView.swift`, `HermesChatCompletionsAPI.swift`, `HermesSecurityUtilities.swift`, `ContentView.swift`) should not gain broad unrelated code.

## Phase 0 Research Summary

See [research.md](./research.md). Key decisions:

1. Use Swift 6 language mode with the locally verified Apple Swift 6.3.3 compiler from Xcode 26.6; keep macOS deployment target at 26.0.
2. Make `project.yml` the source of truth for the `HermesMacOSTest` target and regenerate the checked-in `.xcodeproj`.
3. Prefer default mock-backed Swift tests over live services; live smoke checks are opt-in and documented.
4. Cover complete scope with a named coverage map plus functional and technical suites.
5. Use temporary fixtures and fake credentials for all local runtime, Keychain, TLS, process, Git/SSH, and retention checks.
