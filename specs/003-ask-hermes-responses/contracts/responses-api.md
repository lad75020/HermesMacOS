# Contract: Ask Hermes Responses API

## Profiles

- `GET /v1/profiles` returns profile records with id/name/default/model/provider/supported parameter metadata.
- Ask Hermes uses the selected profile in `X-Hermes-Profile`.
- Empty or unavailable profile selection falls back to `default`.

## Responses Request

- `POST /v1/responses` sends the user prompt, optional attachments, stream flag, optional reasoning effort, previous response ID/session continuation where available, and request ID headers.
- Streaming requests use `Accept: text/event-stream` and parse SSE lines into assistant text, status/event counters, output bubbles, token usage, and errors.
- Non-streaming requests decode the final JSON response envelope.

## Cancellation

- Active work receives an `X-Hermes-Request-Id` generated by HermesMacOS.
- Cancel stops the local task and attempts `POST /v1/requests/{request_id}/cancel`.
- Cancellation must leave the UI in a non-streaming state without deleting prior transcript entries.

## Attachments

- Images are encoded as input image data URLs.
- UTF-8 text/source/config files are inlined with truncation.
- PDFs and Office documents are represented by metadata/description according to existing conversion rules.
- Unsupported or oversized attachments fail before the request is sent.

## Security

- URL validation occurs before adding `Authorization` or other secret-bearing headers.
- Prompt drafts and retained session metadata use redaction/encrypted retention where persisted.
